# FORGED - Demo Video Script (Max 3 Minutes)

> **Note to presenter:** This script is designed to run under the 3-minute limit. Speak at a normal, conversational pace. Ensure your screen recordings do NOT show any prohibited timing/scoring data, athlete NIL, or IOC intellectual property (like Olympic rings). 

---

## 0:00 - 0:15 | Intro & Hook
**[Visual]** 
* Screen recording starts on the FORGED landing page. 
* Clean, dynamic UI with the heading "Find Your Team USA Archetype."

**[Audio / Voiceover]**
"Every fan watching Team USA wonders: 'Where would my body type fit?' For the Team USA x Google Cloud Hackathon, we built FORGED to answer that question. Tackling Challenge 4, The Athlete Archetype Agent, FORGED analyzes 120 years of public biometric data to match fans to historical Team USA athlete archetypes."

## 0:15 - 1:15 | Live Demo (Multimodal Input & Agent Reasoning)
**[Visual]** 
* User clicks "Start Discovery".
* Show the Multimodal input options (Photo, Voice, Form). 
* User selects "Voice" and records a brief description of their height/weight, or uses the Form. 
* Transition to the "Processing" screen. The UI shows the real-time reasoning trace (Server-Sent Events) from the Gemini Agent.
* Reveal the Match: e.g., "The Powerhouse" or "Adaptive Power".

**[Audio / Voiceover]**
"FORGED supports multimodal input—fans can use a photo, voice, or text. Behind the scenes, we use Gemini 3.1 Flash to extract physical traits. 
Next, our backend orchestrated by Gemini 3.1 Pro springs into action. Using function calling, it streams its reasoning trace live via Server-Sent Events, so the user sees exactly how the AI is clustering their traits. 
Here, we’ve been matched with the *Powerhouse* archetype."

## 1:15 - 1:45 | Digital Mirror & Paralympic Parity
**[Visual]** 
* Scroll down to show the "Digital Mirror"—a D3.js visualization plotting the user's data point among historical Team USA athletes.
* Show the Imagen 3.0 generated stylized portrait.
* Highlight the text where conditional phrasing is used: "Your build *could align with*..."
* Navigate briefly to the "Paralympic Explorer" or show the "Adaptive Power" archetype.

**[Audio / Voiceover]**
"The centerpiece is the Digital Mirror, combining a D3 scatter plot with a stylized, non-photorealistic portrait generated by Vertex AI Imagen 3.0. 
Notice the language: to ensure responsible AI, a validation layer enforces conditional phrasing like 'your build *could align with* weightlifting.' 
We also built FORGED with structural Paralympic parity. Olympic and Paralympic data are weighted equally, and we created dedicated, Paralympic-first archetypes with comprehensive classification explainers."

## 1:45 - 2:45 | Under the Hood (Google Cloud & Gemini)
**[Visual]** 
* Quick cut to the Google Cloud Console: Show Cloud Run (where the app is deployed) and BigQuery.
* Briefly show the BigQuery dataset containing the public historical athlete data (ensure NO finish times or NIL are visible).
* Cut to VS Code / AI Studio: Show a snippet of `adk_agent.py` or the Gemini system prompt highlighting the tool calling (e.g., `match_archetype`, `generate_portrait`).

**[Audio / Voiceover]**
"Let’s look under the hood. FORGED is completely serverless, deployed on Google Cloud Run. 
Our biometric data spans 14,000 public records housed in BigQuery. 
The magic happens with Vertex AI. We use Gemini 3.1 Pro as our core agent, executing five specialized tools—from K-means clustering to semantic matching using the `text-embedding-005` model. We also use Vertex AI Context Caching to efficiently reuse our massive archetype corpus across sessions, and Google Search Grounding to connect historical body types with sports featuring at the upcoming LA28 Games."

## 2:45 - 3:00 | Outro
**[Visual]** 
* Back to the FORGED UI: User clicks the "Share Card" button, downloading a sleek LA28-branded social share card.
* Final fade to the FORGED logo and a "Powered by Google Cloud" badge.

**[Audio / Voiceover]**
"FORGED isn't about guaranteeing performance—it's a digital mirror that helps fans see themselves in the collective journey of Team USA through data-driven storytelling. Thank you."

### ⚠️ Pre-Recording Checklist for Compliance:
- [ ] **No Finish Times/Scores:** Ensure your BigQuery demo or UI doesn't show any finish times.
- [ ] **No NIL:** Do not show any real athlete names or images. 
- [ ] **No IOC IP:** Ensure there are no Olympic Rings, Torches, or Paralympic Agitos in your UI.
- [ ] **Correct Terminology:** Double-check that your UI and script say "LA28 Games" (not LA28 Olympics) and "Olympic Games [City] [Year]". 
- [ ] **Conditional Language:** Ensure the UI says "could align with" and NEVER "you would be good at."
